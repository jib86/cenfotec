¿Cómo obtenemos estos datos?

APIs vs Web Scraping

APIApplication Programming Interface — interfaz oficial que un servicio expone para que programas externos lean o escriban datos de forma estructurada y controlada. (oficial)

✓Datos estructurados (JSONJavaScript Object Notation — formato ligero de intercambio de datos con pares clave-valor, ej: {"nombre":"Ana","edad":25}. Estándar de facto en APIs web.)
✓Legal y estable
✓Autenticación clara (API keysClave única que identifica a tu app ante el servicio. Funciona como contraseña para aplicar límites de uso y rastrear solicitudes.)
✗Limitada por rate limitsRestricción sobre cuántas solicitudes puedes hacer por minuto/hora. Ej: la API de X permite ~300 requests cada 15 minutos. y costo
✗No todos los sitios ofrecen API

Web Scraping

✓Funciona con cualquier sitio público
✓Acceso gratuito al contenido
✓Flexible: extraes exactamente lo que necesitas
✗Frágil si el HTMLHyperText Markup Language — lenguaje que define la estructura de páginas web. Cada elemento tiene una etiqueta que BeautifulSoup puede leer y navegar. del sitio cambia
✗Puede violar los Términos de Servicio
BeautifulSoupLibrería de Python que convierte HTML en un árbol de objetos navegable, permitiendo buscar y extraer datos específicos de cualquier página web. SeleniumHerramienta que controla un navegador real (Chrome, Firefox) de forma programática. Útil para sitios con contenido dinámico cargado por JavaScript. ScrapyFramework de Python para web scraping a gran escala. Incluye sistema de colas, concurrencia y exportación automática. Ideal para scrapear miles de páginas.

[Demo] Web Scraping de Productos

Extraemos títulos y precios de libros desde una página web pública de práctica.

python — scraping_productos.py 
import requests
from bs4 import BeautifulSoup

# 1. Descargamos el HTML de la página
url = "https://books.toscrape.com/"
respuesta = requests.get(url)

# 2. Parseamos el HTML con BeautifulSoup
soup = BeautifulSoup(respuesta.text, "html.parser")

# 3. Buscamos todos los artículos de libros
libros = soup.find_all("article", class_="product_pod")

for libro in libros[:5]:
    titulo = libro.h3.a["title"]
    precio = libro.find("p", class_="price_color").text
    rating = libro.p["class"][1]
    print(f"📕 {titulo} | {precio} | ⭐ {rating}")

¿Qué hace cada línea?

requests.get() descarga el HTML como lo haría un navegador.
BeautifulSoup() convierte ese HTML en un árbol navegable.
find_all() busca elementos por selector CSSPatrón que identifica elementos HTML por etiqueta, clase o id. Ej: "article.product_pod" busca todos los <article> con clase "product_pod"..

import pandas as pd

# Construimos una lista de diccionarios
datos = []
for libro in libros:
    datos.append({
        "titulo": libro.h3.a["title"],
        "precio": float(libro.find("p", class_="price_color").text[1:]),
        "rating": libro.p["class"][1]
    })

df = pd.DataFrame(datos)
mapa = {"One":1,"Two":2,"Three":3,"Four":4,"Five":5}
df["rating_num"] = df["rating"].map(mapa)

# Guardamos nuestros datos listos para el análisis
df.to_csv("mis_libros.csv", index=False)
print("✅ DataFrame guardado exitosamente.")

Ética y Legalidad en Scraping y Minería Social

Lo que TODO profesional debe saber

robots.txt y ToS

Siempre revisar robots.txt antes de scrapear. Si un sitio prohíbe scraping en sus Términos de Servicio, no lo hagas.

Rate Limiting

No bombardear un servidor con miles de requests. Agregar pausas (time.sleep(2)) entre solicitudes.

Datos personales y GDPRGeneral Data Protection Regulation — regulación europea de protección de datos. Costa Rica tiene legislación similar (Ley 8968). Aplica a cualquier dato personal.

Nunca recopilar nombres, correos o datos personales sin consentimiento. "Datos públicos" ≠ "datos de uso libre".

Público ≠ libre

Un tweet público NO te autoriza a almacenarlo masivamente, crear perfiles o revenderlo. Verificar copyrightDerecho de propiedad intelectual que protege el contenido original. Los usuarios retienen derechos sobre su contenido aunque sea público. y licencias.

Regla de oro: Si tienes duda sobre si puedes scrapear algo, probablemente no deberías. Usa APIs oficiales cuando existan y siempre anonimiza datos personales.

Análisis de Clickstream

¿Qué hacen los usuarios en tu sitio web?

Un clickstream es la secuencia de clics de un usuario al navegar un sitio. Cada clic registra: la página, el timestampMarca temporal exacta de un evento, ej: "2026-03-12T14:30:05Z". Permite calcular duraciones y ordenar eventos., la duración y de dónde vino.

Navegación típica
¿Qué camino siguen los usuarios más frecuentemente?

Tasa de abandonoBounce rate — porcentaje de sesiones donde el usuario se va tras ver una sola página. Alta tasa = algo no atrae al usuario.
¿En qué página dejan el sitio?

Embudo de conversiónModelo que visualiza el recorrido del usuario hasta la acción deseada (compra). Se llama "embudo" porque en cada paso se pierden usuarios.
¿Cuántos llegan al checkout?

Home Catálogo Producto Carrito Checkout ✓ ← embudo típico

Simulamos datos de navegación y analizamos patrones de conversión.

python — clickstream_analysis.py 
import pandas as pd

clicks = pd.DataFrame({
  "sesion": ["S1","S1","S1","S2","S2","S3","S3","S3","S3","S4"],
  "pagina": ["Home","Catálogo","Producto","Home","Catálogo",
             "Home","Catálogo","Carrito","Checkout","Home"],
  "duracion_seg": [12,45,90,8,20,15,60,30,120,5]
})

print(clicks["pagina"].value_counts())

total = clicks["sesion"].nunique()
conv = clicks[clicks["pagina"]=="Checkout"]["sesion"].nunique()
print(f"Tasa de conversión: {conv/total*100:.0f}%")
Output

Catálogo    3
Producto    1
Carrito     1

Insight: Solo 1 de 4 sesiones completa la compra. S2 y S4 abandonan antes de ver un producto — ¿el catálogo no es atractivo?

Análisis de Redes Sociales como Grafos

Social Network AnalysisDisciplina que estudia las relaciones entre actores usando teoría de grafos y métricas de centralidad para entender la estructura de la red. (SNA)

Las redes sociales son naturalmente grafos: los usuarios son nodos y las interacciones son aristas.

Centralidad de gradoDegree centrality — mide cuántas conexiones tiene un nodo. A más conexiones, mayor centralidad. La métrica más simple de influencia.

¿Quién tiene más conexiones? → Identifica hubs.

Ana ➔ Carlos ➔ Diana
Beto ⬈        ⬉ Luis

Centralidad de intermediaciónBetweenness centrality — cuántas veces un nodo aparece en el camino más corto entre otros. Identifica "puentes" entre comunidades.

¿Quién conecta grupos? → Identifica brokers.

Detección de comunidades

¿Hay subgrupos? → ClústeresGrupos de nodos densamente conectados entre sí pero con pocas conexiones hacia el resto. Equivalente a "grupos de amigos". de usuarios que interactúan frecuentemente.

Ejemplo real: Amazon usa grafos de co-compra: "Clientes que compraron X también compraron Y". Cada producto = nodo, cada co-compra = arista.

[Demo] Grafos con NetworkXLibrería de Python para crear, manipular y estudiar grafos. Incluye algoritmos de centralidad, detección de comunidades, caminos más cortos y visualización.

Construimos un grafo de menciones y encontramos al influenciador principal.

python — social_graph.py 
import networkx as nx

G = nx.DiGraph()
menciones = [
    ("Ana","Carlos"), ("Ana","Luis"),
    ("Beto","Carlos"), ("Beto","Ana"),
    ("Carlos","Diana"), ("Luis","Carlos"),
    ("Diana","Carlos"), ("Eva","Diana"),
]
G.add_edges_from(menciones)

centralidad = nx.in_degree_centrality(G)
top = max(centralidad, key=centralidad.get)
print(f"Influenciador: {top}")
print(f"Centralidad: {centralidad[top]:.2f}")
Output 
Influenciador: Carlos
Centralidad: 0.80

¿Qué significa 0.80? Carlos recibe menciones del 80% del resto de los usuarios participantes, de acuerdo a la fórmula N/N-1. Definitivamente, es la figura central del grupo.

Pipeline: Redes Sociales → Insight

De datos crudos a decisiones de negocio

[Demo] Análisis de Sentimiento con VADER

VADER entiende mayúsculas, emojis y jerga — optimizado para redes sociales.

python — vader_social.py 
from nltk.sentiment import SentimentIntensityAnalyzer
import nltk
nltk.download('vader_lexicon')

sia = SentimentIntensityAnalyzer()

df = pd.DataFrame({'reseñas': [
    "This product is AMAZING!!! Best purchase ever 😍",
    "Terrible experience. Never buying again.",
    "It's okay, nothing special but does the job.",
    "WORST customer service I've EVER dealt with 😡"
]})

df['sentiment_score'] = df['reseñas'].apply(lambda x: sia.polarity_scores(x)['compound'])
print(df[['reseñas', 'sentiment_score']])
Output 
                                         reseñas   sentiment_score
0  This product is AMAZING!!! Best purchase ever 😍    0.92
1         Terrible experience. Never buying again.   -0.54
2     It's okay, nothing special but does the job.    0.07
3    WORST customer service I've EVER dealt with 😡   -0.87

¿Por qué 0.92 y no 0.50? VADER detecta que "AMAZING!!!" con mayúsculas y "!!!" es más positivo que "amazing". El emoji 😍 suma aún más. Integrarlo a Pandas nos prepara para el modelo predictivo final.
