La "Maldición de la Dimensionalidad"

Al trabajar con grandes volúmenes de datos, nos enfrentamos a un fenómeno crítico:

A medida que aumenta el número de características (dimensiones), la cantidad de datos necesarios para generalizar correctamente crece exponencialmente. Demasiadas variables pueden oscurecer los patrones importantes, aumentar el ruido y hacer que los modelos sean computacionalmente ineficientes.

¿Qué es PCA?

Análisis de Componentes Principales

Técnica para reducir dimensiones transformando variables correlacionadas en componentes principales no correlacionados, conservando la información.

Fundamentos Matemáticos

El PCA busca maximizar la varianza proyectada de los datos. Veamos las matemáticas detrás de la magia:

1. Estandarización

PCA es altamente sensible a las escalas de las variables. Se deben normalizar para que todas contribuyan equitativamente, teniendo media \(\mu = 0\) y desviación estándar \(\sigma = 1\) (Z-Score):

$$z = \frac{x - \mu}{\sigma}$$

2. Matriz de Covarianza

Calculamos la Matriz de Covarianza para entender cómo las variables varían juntas respecto a la media. Esto captura las relaciones lineales entre todas las pares de variables:

$$Cov(X, Y) = \frac{\sum_{i=1}^{n} (X_i - \bar{X})(Y_i - \bar{Y})}{n-1}$$

3. Eigenvectors y Eigenvalues

Resolvemos la ecuación característica para la matriz de covarianza \(A\):

$$det(A - \lambda I) = 0$$

Donde \(\lambda\) son los eigenvalues e \(I\) es la matriz identidad.

  • Eigenvectors (Autovectores): Vectores que no cambian de dirección durante una transformación lineal. Indican la dirección de los nuevos ejes (componentes principales).
  • Eigenvalues (Autovalores): Escalares asociados a los autovectores. Indican la magnitud de la varianza capturada en esa dirección específica. A mayor eigenvalue, mayor información retiene ese componente.

Algoritmo PCA (Paso a Paso)

Entrada: Matriz de datos \(X\) de dimensiones \(n \times m\).

01

Centrado de Datos

¿Qué hacemos?: Restamos el promedio a cada variable.
¿Qué obtenemos?: Movemos la nube de puntos al origen (0,0) para analizar puramente la variabilidad, sin que afecte la posición de los datos.

02

Matriz de Covarianza

¿Qué hacemos?: Medimos cómo varía cada variable con respecto a las demás.
¿Qué obtenemos?: Un mapa completo de las relaciones lineales (correlaciones) entre todas nuestras variables.

03

Descomposición (Eigenvalues)

¿Qué hacemos?: Descomponemos la matriz de relaciones en direcciones y fuerzas.
¿Qué obtenemos?: Identificamos los ejes principales donde está la mayor información (varianza) y descartamos el ruido.

04

Proyección Final

¿Qué hacemos?: Multiplicamos nuestros datos originales por los componentes seleccionados.
¿Qué obtenemos?: Una nueva versión simplificada de los datos (menos columnas) que preserva la información más importante.

Complejidad Computacional: Generalmente \(O(m^2 n + m^3)\), donde:
\(n\) = número de muestras (filas) | \(m\) = número de variables (columnas).

pca_pipeline.py
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns
from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler

# 1. Carga y Preparación (Simulación)
data = {
    'monto_compra': [100, 150, 120, 300, 450, 110, 500, 1000],
    'frecuencia_mensual': [2, 3, 2, 5, 8, 2, 9, 15],
    'tiempo_en_sitio': [10, 15, 12, 30, 45, 11, 50, 120],
    'paginas_vistas': [5, 7, 5, 12, 20, 4, 25, 40]
}
df = pd.DataFrame(data)

# 2. Estandarización (Obligatorio)
# Ponemos todas las variables en la misma escala
scaler = StandardScaler()
df_scaled = scaler.fit_transform(df)

# 3. Aplicación de PCA 
pca = PCA(n_components=2) 
pca_results = pca.fit_transform(df_scaled)

# 4. DataFrame de resultados (Espacio Latente)
df_pca = pd.DataFrame(data=pca_results, columns=['PC1', 'PC2'])
print(f"Varianza explicada: {pca.explained_variance_ratio_}")

Justicia Matemática (Z-Score)

Si comparas Salarios (millones) con Edad (decenas), el algoritmo "escuchará" solo al dinero porque el número es más grande.

La estandarización pone a todos en la misma balanza para que cada variable cuente por su comportamiento, no por su tamaño.

El Concepto de "Scree" (Escombros)

En geología, el "scree" son las rocas sueltas y escombros que se acumulan en la base de un acantilado.

En PCA, el gráfico de varianza (eigenvalues) siempre tiene forma de montaña:

  • El Acantilado (Señal): Los primeros componentes que capturan mucha información (pendiente pronunciada).
  • Los Escombros (Ruido): Los componentes finales que aportan muy poco (pendiente plana).

La Regla del Codo

Cortamos donde termina el acantilado y empiezan los escombros.

×

PC1: El Índice de Magnitud

El Primer Componente Principal (PC1) siempre es el "Rey de la Varianza". Captura la mayor cantidad de información posible en una sola línea.

¿Qué significa en Negocios?

Casi siempre representa el "Tamaño General" del cliente.

  • Valores Altos: Clientes "Gigantes" (Gastan mucho, visitan mucho, compran mucho).
  • Valores Bajos: Clientes "Pequeños" o inactivos.

Analogía Visual

Imagina que pesas a tus clientes.
PC1 te dice cuánto pesan en total, sin importar si es músculo o grasa.

×

PC2: Dimensión de Comportamiento

El Segundo Componente (PC2) captura lo que PC1 ignoró. Es la "personalidad" o el "estilo" del dato.

¿Qué significa en Negocios?

Distingue a clientes que tienen el mismo "Tamaño" (PC1) pero actúan diferente.

"Dos clientes gastan $1000 al mes (mismo PC1). Pero uno hace 1 compra de $1000 y el otro 100 compras de $10. PC2 captura esa diferencia."

Analogía Visual

PC1 es el Volumen del paquete.
PC2 es la Forma del paquete (largo y fino vs. cuadrado y compacto).
