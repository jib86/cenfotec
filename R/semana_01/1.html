Tema 1 — Fundamentos del Entorno

¿Qué es R y RStudio?

R

(El Motor Analítico)

Imagina a R como el motor de alto rendimiento de un vehículo. Es un lenguaje de programación interpretado (ejecuta instrucciones directamente línea por línea sin requerir compilación previa) diseñado específicamente por y para estadísticos. Su propósito central es la manipulación masiva de datos, el cálculo complejo y la generación de gráficos con calidad de publicación.

Arquitectura Base

Basado en el antiguo lenguaje S, su filosofía es elegante: "Todo lo que existe en R es un objeto" (datos, funciones, modelos) y "Todo lo que ocurre es la llamada a una función".

RStudio

(El Panel de Control)

Si R es el motor, RStudio es el chasis y la cabina del piloto. Es un Entorno de Desarrollo Integrado (IDE). Por sí solo no realiza cálculos; su función es enviar tus instrucciones al motor (R) de forma amigable, organizando tu código, gráficos y memoria en una interfaz visual.

Los 4 Paneles Principales
  • Source/Script: Tu bloc de notas. Donde redactas y guardas tus instrucciones.
  • Consola: El intérprete. Ejecuta el código inmediatamente, pero no lo guarda a largo plazo.
  • Environment: El inventario de la RAM. Muestra todas las variables y tablas cargadas.
  • Files/Plots: Visor de tus carpetas locales y lienzo donde se pintan los gráficos.
Tema 1 — Formatos de Trabajo

El Script Clásico vs. R Markdown

Scripts (.R)

Es el archivo tradicional de programación. Son meros archivos de texto plano diseñados para ser leídos por la máquina. Solo admiten código R puro y apuntes breves comentados tras un símbolo #. Su propósito es ejecutar secuencias de tareas automatizadas en silencio (ej. limpiar datos a medianoche).

# Cálculo simple sin ninguna narrativa explicativa
# Difícil de leer para un tomador de decisiones
ventas <- c(150, 200, 310)
mean(ventas)

R Markdown (.Rmd)

El estándar de oro para reportes de Data Science. Utiliza el paradigma de "Programación Literaria": no escribes código con comentarios, escribes un ensayo explicativo en lenguaje humano e incrustas "islas" de código vivo (Chunks). Al final, R "teje" un reporte unificado (HTML, PDF o Word) que mezcla texto pulido con resultados, tablas y gráficos.

Estructura Tríptico de un .Rmd
  1. YAML Header (El Pasaporte): Metadatos encerrados entre ---. Dicta las "reglas del juego": Título, Autor y el formato estético de salida.
  2. Markdown (La Narrativa): Todo el texto argumentativo, formulado libremente asumiendo que el lector final no sabe programación.
  3. Chunks (Los Motores de Datos): Bloques de código reales iniciados con ```{r} y cerrados con ```. Proveen los números duros a la narrativa.
Tema 1 — El Lenguaje de R

Operaciones Matemáticas y Lógicas

Para manipular bases de datos, necesitas transformar columnas matemáticamente (ej. Ingresos - Gastos) o filtrarlas bajo condiciones específicas (ej. Mostrar solo "Ventas > 1000").

Aritméticas

Para cálculos numéricos estándar.

  • + - Suma y Resta
  • * / Mult. y División
  • ^ Exponenciación (Potencia)
  • %% Módulo (Resto de división)

Relacionales

Comparan valores. Retornan TRUE o FALSE.

  • == Igualdad (Exactamente igual a)
  • != Desigualdad (Diferente de)
  • < > Menor o Mayor estricto
  • <= Menor o igual ( / Mayor o igual)

Booleanos

Combinan múltiples condiciones para filtrar.

  • & AND (Ambas deben ser TRUE)
  • | OR (Al menos una debe ser TRUE)
  • ! NOT (Invierte: TRUE pasa a FALSE)
El Cuidado del Doble Símbolo:
Un solo & filtra columnas enteras elemento por elemento. El doble && solo evalúa el primer registro (usado en control de flujo `if`).

Comportamiento Nativo: La Regla de Reciclaje

A diferencia de Excel donde multiplicar columnas de diferente tamaño da error, R "recicla" invisiblemente los datos. Si sumas una columna de 10 datos con una de 2, R repetirá los 2 datos del vector corto 5 veces consecutivas para obligar a que cuadren. ¡Cuidado con no notarlo!

Tema 1 — La Materia Prima

Los 3 Tipos de Datos Básicos

Antes de construir estructuras complejas, necesitamos entender de qué están hechas. Todo en R se reduce a tres bloques atómicos.

Numéricos

Cualquier valor matemático. Por defecto en R, todos los números son tratados como continuos (con decimales), incluso si escribes un entero.

Ejemplos:
15.5 # Un salario o peso
42 # Una edad

Textos (Strings)

Letras, palabras o frases (Incluso números si no quieres operarlos matemáticamente, como un ID). Obligatoriamente deben ir encerrados entre comillas.

Ejemplos:
"Costa Rica"
"Cliente_VIP_001"

Lógicos (Booleans)

Interruptores binarios. Le indican a la computadora si una condición es cierta o falsa. Son el núcleo de la toma de decisiones algorítmicas (Filtros).

Ejemplos:
TRUE # Encendido / Sí
FALSE # Apagado / No
Tema 1 — Ingesta de Información

¿Cómo meto mi Excel a R?

R trabaja procesando datos en la Memoria RAM (Environment). Para analizar información del mundo real, primero debemos "leer" el archivo desde el disco duro y "asignarlo" (<-) a una variable en la memoria.

Archivos CSV

(Valores Separados por Comas)

Piensa en ellos como un archivo de texto simple donde las columnas están separadas por "comitas". Son universales, súper ligeros y a R le encantan.

¿Por qué usarlos?

  • Máxima compatibilidad: Cualquier programa del mundo puede abrirlos.
  • Cero "basura": No guardan colores, ni negritas, ni fórmulas. Solo datos puros.
  • Velocidad: R los lee muchísimo más rápido que un Excel.
Haz clic para ver el código

Archivos CSV

¡Vienen listos para usarse! R base los lee de forma nativa sin que tengamos que instalar nada extra.

# Usamos la función nativa read.csv()
mis_datos <- read.csv("reporte.csv")

Nota: El símbolo <- significa que estamos guardando el archivo en la RAM con el nombre "mis_datos".

Archivos Excel

(.xlsx o .xls)

El formato corporativo tradicional. A diferencia de los ligeros CSV, los Excel son archivos pesados llenos de colores, fórmulas y múltiples pestañas.

Los Desafíos (Banderas Rojas)

  • No son datos puros: Tienen celdas combinadas, gráficos flotantes y macros que a R no le interesan.
  • Múltiples "Hojas": R necesita saber exactamente de de qué pestaña (Hoja) quieres extraer la tabla.
  • Solución: Hay que "limpiarlos" mentalmente antes de pasarlos a R.
Haz clic para ver cómo leerlos

Archivos Excel

R originalmente no entiende el "idioma Microsoft". Por eso, primero debemos enseñarle instalando un "traductor" (un paquete oficial llamado readxl).

# 1. Cargamos el traductor a la RAM
library(readxl)

# 2. Leemos el archivo usando el traductor
mis_datos <- read_excel("reporte.xlsx")

Nota: En R, descargar paquetes creados por otras personas para resolver problemas específicos es lo más natural del mundo.

Tema 1 — Ecosistema de R

Paquetes y Estructuras de Datos

Paquetes (Librerías)

R Base viene con lo esencial. Los paquetes son como "Aplicaciones" (ej. WhatsApp, Excel) que desarrolladores externos crean para dar superpoderes a R (ej. Machine Learning, gráficos web).

Paso 1: Instalación (Una sola vez)

Descarga el código de internet a tu disco duro.

install.packages("nombre_paquete")
Paso 2: Carga (En cada sesión)

Abre la "App" en tu memoria RAM para poder usarla hoy.

library(nombre_paquete)
Las 4 Piezas de Lego Fundamentales

1. Vectores

1D, Homogéneos

Una fila o columna simple. Sus elementos deben ser del mismo tipo (todo texto o todo número).

Jerarquía de Coerción: Si mezclas textos y números, R convierte los números a texto forzosamente para no fallar.

2. Matrices

2D, Homogéneas

Una tabla matemática de 2 dimensiones construida con vectores ensamblados. Al ser homogénea, solo sirve si toda la tabla es numérica o toda es texto. Se usa en álgebra lineal.

3. Listas

1D, Heterogéneas

El comodín de R. Es un contenedor universal donde cada "casilla" puede guardar un objeto distinto: en la posición 1 un número, en la 2 un texto, en la 3 ¡un gráfico entero!

4. Data Frames

2D, Heterogéneos

La equivalencia en código a una hoja de Excel. Son listas especiales donde cada elemento (columna) es un vector, y todos los vectores miden lo mismo (filas). Permiten mezclar columnas numéricas con columnas de texto.

Tema 1 — Filosofía del Código

Programación Funcional en R

R no se centra en decirle a la máquina cómo hacer las cosas paso a paso, sino en qué evaluar. Las funciones son "ciudadanos de primera clase".
(Significa que una función puede guardarse en un objeto, pasarse como ingrediente a otra función, ¡e incluso ser el resultado matemático de otra función!).

Anatomía de una Función

Las funciones encapsulan tareas repetitivas. Tienen un nombre, reciben argumentos (ingredientes), procesan el código interno, y retornan el plato final.

limpiar_datos.R
mi_funcion <- function(arg1, arg2 = valor_por_defecto) {
    resultado <- arg1 + arg2
    return(resultado) # Retorna última evaluación
}

La familia apply

El problema: Los bucles for clásicos son lentos, verbosos y difíciles de leer cuando analizas millones de datos.

La solución: Es como pasar de lavar platos a mano (uno por uno) a usar un lavavajillas industrial (todos a la vez). Es la forma nativa de R para ejecutar operaciones a gran escala.

Haz clic para ver cómo se usa

La familia apply

Toma una función (un verbo) y se la aplica instantáneamente a una lista de objetos (sujetos), sin bucles.

lapply(lista_archivos, promedio)

(Lee: "Aplica la función promedio a CADA archivo simultáneamente").

Regla de Oro de lapply Devolverá siempre una Lista.
Tema 1 — Práctica Guiada

Aterrizando Conceptos

Veamos R en acción. Cargamos un dataset (mtcars) y creamos un modelo de IA (Regresión Lineal Simple) para predecir el consumo de gasolina según el peso.

modelo_prediccion_autos.R
# 1. Cargar paquete gráfico (Motor de visualización)
library(ggplot2)

# 2. Cargar datos (mtcars viene preinstalado)
# ↓ ATENCIÓN: La flecha significa "ASIGNAR". Tomamos 'mtcars' y lo guardamos dentro de la variable 'autos_data'
autos_data <- mtcars
head(autos_data, 3) # Ver primeras 3 filas para explorar formato

# 3. Entrenar el Modelo Predictivo (Regresión Lineal Simple)
# Fórmula (Y ~ X): Predecir 'mpg' (Millas/Galón) usando 'wt' (Peso)
mi_modelo <- lm(formula = mpg ~ wt, data = autos_data)

# 4. Interpretar la Inteligencia creada
summary(mi_modelo) # Revela matemática pura si el Peso afecta el consumo

Asignación <-

En R, la flecha significa "asignar a". Es como meter datos del lado derecho en una caja con el nombre del lado izquierdo. Guarda el resultado devuelto en la memoria RAM del sistema.

Función lm()

Acrónimo de Linear Model. Calcula la línea matemática de mejor ajuste a través de los datos.

Fórmula ~

La tilde (virgulilla) separa la Variable Objetivo que quieres predecir (Y) de los parámetros Predictores (X).
