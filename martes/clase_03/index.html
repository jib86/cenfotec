Estrategias de Limpieza

Tratamiento sistemático de problemas comunes en datasets:

Imputación ¿Huecos?

Problema: La mayoría de modelos matemáticos colapsan si encuentran un espacio vacío (NaN).

Solución: Rellenar ese hueco con un valor inteligente.

  • Media/Promedio: Úsala si tus datos son "normales".
  • Mediana: Úsala si tienes datos locos (outliers) que arruinarían el promedio (ej. Salario de Elon Musk).
  • Moda: Úsala para texto o categorías (ej. rellenar con el color más común).

Normalización Escalas

Problema: Si tienes "Edad (0-100)" y "Sueldo (0-1,000,000)", el modelo le dará toda la importancia al Sueldo solo porque el número es más grande.

Solución: Aplastar todos los números para que jueguen en la misma cancha.

  • Min-Max: Obliga a todos a estar entre 0 y 1.
  • StandardScaler: Transforma los datos para que su centro sea 0. Ideal si usas modelos complejos (SVM, Logística).

Estrategias de Limpieza

Transformación de datos para el modelo:

Encoding Categóricos

Convertir texto a tensores numéricos.

  • One-Hot: Crea columnas binarias por categoría. (Rojo → [1,0,0]). Ideal si no hay orden.
  • Label Encoding: Asigna enteros (Rojo→1, Azul→2). Riesgo: El modelo puede interpretar falsamente que 2 > 1 (Azul es "mejor" que Rojo), creando una jerarquía inexistente.

Outliers Anomalías

Detección y tratamiento.

  • IQR Method: Rango intercuartil.
  • Z-Score > 3: Desviación estándar.
  • Tratamiento: Clip (topar), eliminar o transformar logarítmicamente.

Responsabilidad y Sesgos

Los algoritmos no son neutros; aprenden los prejuicios de los datos.

Sesgos Comunes

  • Representación: El dataset de entrenamiento no representa a la población real (ej. un sistema de reconocimiento facial entrenado solo con hombres blancos fallará con otras demografías).
  • Histórico: Los datos reflejan prejuicios humanos del pasado. Si usamos datos históricos de contratación discriminatorios, el modelo aprenderá a discriminar automáticamente.
  • Medición y Variables Proxy: Ocurre al medir mal una característica. Una "variable proxy" es un dato indirecto usado cuando el real no está disponible (ej. usar "Código Postal" como proxy de "Riesgo Crediticio" puede esconder un sesgo racial/económico grave).

Transparencia (XAI)

XAI (Explainable Artificial Intelligence): Inteligencia Artificial Explicable. Busca hacer que los resultados sean comprensibles para humanos, evitando "cajas negras".

  • SHAP (SHapley Additive exPlanations): Asigna un valor de importancia a cada variable (juego cooperativo) para explicar por qué el modelo predijo X.
  • LIME (Local Interpretable Model-agnostic Explanations): Explica una predicción individual aproximándola con un modelo lineal simple y local.

Paso 3: Transformación

El núcleo del preprocesamiento: Limpiar y Escalar.

  • Usamos Mediana para edad (robusto a outliers).
  • Usamos Z-Score para ingresos (mejor para modelos lineales).
Tip Crítico (Data Leakage):

Siempre ajuste el scaler (.fit()) SOLO con los datos de entrenamiento (X_train).

Si haces .fit() con todo el dataset, el modelo "aprenderá" estadísticas (media, varianza) de los datos de test, haciendo trampas. Esto invalida tus métricas de evaluación.

preprocessing.py 
# A. Imputación: Rellenar huecos con Mediana (robusta a outliers)
imputer = SimpleImputer(strategy='median')
df['edad'] = imputer.fit_transform(df[['edad']])

# B. Normalización (Ingresos)
scaler = StandardScaler()
df['ingreso_s'] = scaler.fit_transform(df[['ingresos']])

# Verificación rápida
print(df[['edad', 'ingreso_s']].describe())
