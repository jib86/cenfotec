¿Como resolver un Sudoku... sin adivinar?

Enfoque humano (fuerza brutaFuerza Bruta|||Estrategia que prueba todas las combinaciones posibles hasta encontrar la solucion correcta. Para un Sudoku 9x9 con 30 celdas vacias, eso significaria probar hasta 9^30 combinaciones — un numero astronomico. Es como intentar abrir una caja fuerte probando todos los codigos uno por uno.)

Imagina que intentas resolver un Sudoku probando cada numero del 1 al 9 en cada celda vacia, sin pensar. Si el numero genera un conflicto, lo borras y pruebas el siguiente:

Fuerza bruta: pongo un 1 en la primera celda vacia, verifico si hay conflicto en la fila, columna o bloque 3x3. Si lo hay, pruebo con 2, luego 3... y asi con cada celda. ¡Puede tomar millones de intentos!
# Pseudocodigo: fuerza bruta (muy lento) for cada_celda_vacia: for n in 1..9: if no_hay_conflicto(celda, n): colocar(n) else: retroceder() # backtrack

Problema: con 30 celdas vacias, hay hasta 9^30 combinaciones posibles. Es un numero con 29 digitos — ni la computadora mas rapida del mundo lo resolveria en un tiempo razonable con fuerza bruta pura.

Enfoque IA (CSPCSP — Constraint Satisfaction Problem|||Tipo de problema donde debes asignar valores a variables respetando un conjunto de restricciones. En vez de buscar a ciegas, el CSP descarta opciones imposibles antes de probar — reduciendo drasticamente el espacio de busqueda. Sudoku, horarios y mapas son CSP clasicos.)

En vez de adivinar, definimos las reglas del juego y dejamos que un algoritmo inteligente descarte lo imposible antes de probar:

CSP: cada celda es una variable con dominio {1..9}. Las restricciones dicen "no repetir en fila, columna ni bloque 3x3". El solver propaga restricciones y reduce los dominios automaticamente — si pongo un 5 en una celda, el propagador elimina el 5 de todas las celdas de la misma fila, columna y bloque.
% CSP: definir variables y restricciones Variables: X11, X12, ... X99 Dominios: cada Xij ∈ {1..9} Restricciones: all_different(fila_i) % ∀ fila all_different(columna_j) % ∀ columna all_different(bloque_k) % ∀ bloque 3x3

El solver encuentra la solucion en milisegundos porque la propagacion de restriccionesPropagacion de Restricciones|||Tecnica que, al asignar un valor a una variable, elimina automaticamente valores imposibles de las variables vecinas. Si pongo un 5 en una celda, el propagador quita el 5 del dominio de todas las celdas en la misma fila, columna y bloque. Esto reduce drasticamente lo que hay que probar. reduce el espacio de busqueda de miles de millones a unas pocas opciones.

La idea clave de hoy: En vez de buscar soluciones a ciegas, vamos a aprender a modelar problemas como CSP y dejar que algoritmos inteligentes hagan el trabajo pesado. Esta tecnica se usa en la vida real para programar horarios, rutas de transporte, configuracion de productos y mucho mas.

Repaso: ¿Que aprendimos la semana pasada?

Q1 Recursividad en Prolog

¿Como repite Prolog una accion muchas veces, si no tiene bucles como for o while?

Respuesta: En Prolog la unica forma de repetir es la recursividad: un predicado que se llama a si mismo con datos cada vez mas pequenos hasta llegar al caso base (la condicion de parada). Ejemplo: para recorrer una lista, el predicado procesa la cabeza y se llama recursivamente con la cola, hasta que la lista queda vacia.
% Caso base: lista vacia, sumatoria = 0 suma([], 0). % Caso recursivo: sumar H + la suma del resto suma([H|T], S) :- suma(T, S1), S is H + S1.

Q2 El operador Cut

¿Para que sirve el simbolo ! en Prolog y cuando lo usamos?

Respuesta: El ! (cut) le dice a Prolog: "ya encontraste la respuesta correcta, no busques mas alternativas". Elimina los puntos de backtracking. Se usa para evitar respuestas duplicadas o incorrectas, y para mejorar el rendimiento cuando sabemos que la primera solucion es suficiente.
% Sin cut: Prolog podria dar respuestas redundantes max(X, Y, X) :- X >= Y, !. % si X >= Y, X es max. ¡Punto! max(_, Y, Y). % si no, Y es max

Q3 Aritmetica: = vs is

¿Cual es la diferencia entre X = 3+4 y X is 3+4?

Respuesta: X = 3+4 hace unificacion — guarda la estructura literal "3+4" sin calcular. X is 3+4 hace evaluacion aritmetica — calcula y produce X = 7. Siempre usa is cuando necesites un resultado numerico.

Q4 findall/3

¿Que hace findall(X, Cond, Lista)?

Respuesta: findall(X, Condicion, Lista) ejecuta la Condicion todas las veces posibles y recolecta cada resultado X en una Lista. Es como un "SELECT * WHERE" de bases de datos — reune todas las respuestas en una sola lista.
gato(tom). gato(felix). gato(garfield). ?- findall(X, gato(X), L). % L = [tom, felix, garfield]

Conexion con hoy: El backtracking que Prolog hace automaticamente es la base de los algoritmos CSP. Hoy veremos como mejorarlo dramaticamente con propagacion de restricciones.

Bloque B · Definicion Fundamental

¿Que es un CSP? (Constraint Satisfaction Problem)

Analogia: Imagina que organizas los asientos de una boda. Cada invitado (variable) debe sentarse en una mesa (dominio), pero hay reglas: la tia Maria no puede estar con el tio Juan (restriccion), los novios deben estar en la mesa principal (restriccion), etc. Un CSP es exactamente eso: asignar valores respetando reglas.

Un CSPCSP — Constraint Satisfaction Problem|||Un Problema de Satisfaccion de Restricciones se define por tres componentes: (1) un conjunto de variables, (2) un dominio de valores posibles para cada variable, y (3) un conjunto de restricciones que limitan que combinaciones de valores son permitidas. El objetivo es encontrar una asignacion que satisfaga TODAS las restricciones simultaneamente. es un problema matematico definido por exactamente tres cosas:

Variables (X)

Las incognitas a las que debemos asignar un valor. En la boda: cada invitado. En Sudoku: cada celda vacia. En horarios: cada curso.

Dominios (D)

Los valores posibles que puede tomar cada variable. En la boda: las mesas disponibles. En Sudoku: {1..9}. En horarios: las franjas horarias.

Restricciones (C)

Las reglas que limitan que combinaciones son validas. En la boda: "estos dos no juntos". En Sudoku: "no repetir en fila/columna".

Un CSP esta RESUELTO cuando:

Cada variable tiene un valor asignado de su dominio (asignacion completaAsignacion Completa|||Una asignacion donde todas las variables del CSP tienen un valor de su dominio. Si ademas cumple todas las restricciones, se llama asignacion consistente o solucion.) Y todas las restricciones se cumplen simultaneamente.

Diferencia con optimizacion:

Un CSP no busca la "mejor" solucion, solo una solucion valida. No hay funcion de costo ni puntuacion — solo "cumple todas las reglas" o "no cumple".

Bloque B · ¿Donde se usan?

CSP en el mundo real: no es solo teoria

Los CSP se usan en industrias reales todos los dias. Cada vez que hay que asignar recursos cumpliendo reglas, hay un CSP detras:

Horarios universitarios

Variables: Cada seccion de curso

Dominios: Combinaciones de (aula, franja horaria)

Restricciones: Un profesor no puede estar en dos aulas a la vez, un aula no tiene dos clases simultaneas, ciertos cursos requieren labs...

Rutas de transporte

Variables: Cada vehiculo y su ruta

Dominios: Posibles secuencias de entregas

Restricciones: Capacidad del vehiculo, ventanas de tiempo de entrega, distancia maxima...

Configuracion de productos

Variables: Cada componente del producto (CPU, RAM, disco...)

Dominios: Opciones disponibles para cada componente

Restricciones: Compatibilidad entre componentes, presupuesto maximo, requisitos minimos...

Sudoku y puzzles

Variables: Cada celda vacia

Dominios: {1, 2, 3, 4, 5, 6, 7, 8, 9}

Restricciones: No repetir numero en misma fila, columna, ni bloque 3x3

¿Por que aprender CSP? Porque el patron "asignar valores cumpliendo reglas" aparece en casi cualquier dominio. Si sabes modelar un problema como CSP, puedes usar solvers existentes (como CLP(FD) en Prolog) en vez de programar la busqueda a mano. Es como tener un "resolvedor universal" de problemas combinatorios.

Bloque B · Ejemplo Clasico

Coloreo de Mapas: aprendamos a modelar

El problema en palabras simples

Tenemos un mapa con regiones. Queremos pintar cada region con un color, pero dos regiones que comparten frontera no pueden tener el mismo color. ¿Se puede hacer con solo 3 colores?

Paso a paso: modelar como CSP

Paso 1 — Identificar las variables: Cada region del mapa es una variable. Si el mapa tiene 5 regiones (A, B, C, D, E), entonces tenemos 5 variables.
Variables: {A, B, C, D, E} — una por cada region
Dominios: Cada variable ∈ {rojo, verde, azul} — los 3 colores
Restricciones: Si A y B son vecinos → A ≠ B (colores diferentes)

En Prolog (generate-and-test)

¿Que hace este codigo? Define los 3 colores como hechos. Luego, colorear/5 asigna un color a cada variable y verifica que vecinos sean diferentes. Prolog prueba combinaciones automaticamente via backtracking.
% Paso 1: definir el dominio (colores disponibles) color(rojo). color(verde). color(azul). % Paso 2: asignar y verificar restricciones colorear(A,B,C,D,E) :- color(A), color(B), color(C), color(D), color(E), % Restricciones: vecinos distintos A \= B, A \= C, B \= C, B \= D, C \= D, C \= E, D \= E. ?- colorear(A,B,C,D,E). % A=rojo, B=verde, C=azul, D=rojo, E=verde

Problema de este enfoque: Prolog primero genera TODAS las asignaciones (generate-and-testGenerate-and-Test|||Estrategia naive donde primero se generan todas las combinaciones posibles y luego se prueban contra las restricciones. Con 5 variables y 3 colores son 3^5 = 243 combinaciones. Para problemas grandes esto es inaceptablemente lento. Los algoritmos CSP mejoran esto descartando valores imposibles antes de generar.) y luego verifica. Con 5 variables son 243 combinaciones — manejable. Pero con 50 variables serian 3^50. ¡Necesitamos algo mejor!

Bloque B · Clasificacion

Tipos de Restricciones: ¿cuantas variables involucran?

Las restricciones se clasifican segun cuantas variables afectan. Esto es importante porque determina que tecnicas de resolucion podemos aplicar. Pensalo asi: una regla que solo afecta a un invitado es mas facil de resolver que una que afecta a toda una mesa.

UnariasRestriccion Unaria|||Restringe el dominio de una sola variable. Ejemplo: "X ≠ rojo" elimina "rojo" del dominio de X directamente. Son las mas simples y se resuelven primero, reduciendo el dominio antes de considerar restricciones mas complejas. (1 variable)

Afectan a una sola variable. Se resuelven simplemente eliminando valores del dominio.

Analogia: "El invitado Pedro es vegetariano" → solo le sirves platos sin carne. Afecta solo a Pedro.
% "X no puede ser rojo" X \= rojo % Resultado: quitar "rojo" de D(X) % "La temperatura debe ser > 0" T > 0 % Resultado: quitar valores ≤ 0 de D(T)

BinariasRestriccion Binaria|||Relaciona exactamente dos variables. Ejemplo: "A ≠ B" (dos regiones vecinas deben tener colores diferentes). Son las mas comunes y las que se analizan con consistencia de arco (AC-3). Todo CSP puede transformarse a uno con solo restricciones binarias. (2 variables)

Relacionan exactamente dos variables. Son las mas comunes en la practica.

Analogia: "Pedro y Maria no pueden sentarse en la misma mesa" → una regla que conecta a dos personas.
% "A y B deben ser diferentes" A \= B % "X debe ser menor que Y" X < Y

GlobalesRestriccion Global|||Involucra tres o mas variables simultaneamente. Ejemplo: all_different([X1,X2,X3,X4]) exige que todas sean diferentes entre si. Son mas expresivas y permiten algoritmos de propagacion mas potentes que las restricciones binarias individuales. (3+ variables)

Involucran 3 o mas variables a la vez. Mas expresivas y eficientes.

Analogia: "Todos los invitados de la mesa 1 deben hablar idiomas diferentes" → una regla que afecta a todo un grupo.
% "Todas deben ser diferentes entre si" all_different([X1,X2,X3,X4]) % "La suma de todas = 100" sum([X1,X2,X3], #=, 100)

Dato clave: Cualquier CSP con restricciones globales puede transformarse en uno equivalente con solo restricciones binarias (introduciendo variables auxiliares). Por eso muchos algoritmos se enfocan en restricciones binarias — pero las globales son mas eficientes en la practica porque los solvers tienen algoritmos especializados para ellas.

Permite ver la estructura del problema de un vistazo: cuantas conexiones tiene cada variable, que grupos estan mas acoplados, etc. Es como un mapa de relaciones.. ¿Por que es util? Porque te permite ver de un vistazo la estructura del problema: que variables estan mas conectadas, donde hay mas conflictos potenciales.

Ejemplo: Mapa de 4 regiones

Bloque C · Tecnica de Consistencia #1

Consistencia de Nodo: el primer filtro

Idea clave: Antes de empezar a buscar soluciones (que es costoso), podemos reducir los dominios eliminando valores que seguro no van a funcionar. Es como filtrar candidatos antes de una entrevista: si el puesto requiere ingles, descartamos de entrada a quienes no hablan ingles.

Consistencia de NodoConsistencia de Nodo (NC)|||Un nodo (variable) es consistente si todos los valores de su dominio satisfacen las restricciones unarias de esa variable. Es el filtro mas basico: eliminamos valores que violan reglas de una sola variable, antes de considerar relaciones entre variables. (Node Consistency)

Un nodo (variable) es consistente si todos los valores de su dominio cumplen las restricciones unarias (las que solo afectan a esa variable).

¿Que hacemos? Miramos cada variable individualmente. Si tiene una restriccion que solo la afecta a ella, eliminamos los valores que la violan. Esto se hace antes de cualquier busqueda.

Ejemplo paso a paso

Tenemos X con D(X) = {1, 2, 3, 4, 5} y la restriccion X > 2.

Verificamos cada valor:

X=1: ¿1 > 2? NO → eliminar X=2: ¿2 > 2? NO → eliminar X=3: ¿3 > 2? SI → se queda X=4: ¿4 > 2? SI → se queda X=5: ¿5 > 2? SI → se queda

Resultado: D(X) = {3, 4, 5} — ¡redujimos de 5 a 3 opciones!

¿Y las restricciones binarias?

La consistencia de nodo solo maneja restricciones unarias. Para restricciones entre dos variables (como A ≠ B) necesitamos algo mas poderoso: la consistencia de arco.

Consistencia de ArcoConsistencia de Arco (AC)|||Un arco (X → Y) es consistente si para cada valor de X existe al menos un valor de Y que satisface la restriccion entre ellos. Si algun valor de X no tiene ningun "soporte" en Y, se elimina del dominio de X. (Arc Consistency)

Un arco X → Y es consistente si: para cada valor posible de X, existe al menos un valor de Y que cumple la restriccion entre ellos.

En palabras simples: para cada opcion de X, me pregunto: "¿existe alguna opcion de Y que no genere conflicto?". Si la respuesta es NO para algun valor de X, ese valor se elimina del dominio de X. Llamamos "soporte" a ese valor de Y compatible.

Ejemplo de consistencia de arco

D(X) = {1, 2, 3}, D(Y) = {1, 2, 3}. Restriccion: X < Y.

X=1: ¿∃ Y > 1? → Y=2 ✓ o Y=3 ✓ → tiene soporte X=2: ¿∃ Y > 2? → Y=3 ✓ → tiene soporte X=3: ¿∃ Y > 3? → NO HAY → eliminar X=3

Resultado: D(X) = {1, 2}

Bloque C · Algoritmo AC-3

AC-3: Consistencia de Arco Sistematica

El algoritmo AC-3AC-3|||Algoritmo que aplica consistencia de arco a todos los arcos del CSP de forma sistematica. Usa una cola de arcos pendientes. Cuando un dominio se reduce, vuelve a revisar los arcos afectados. Complejidad: O(e·d³) donde e = arcos y d = tamanio maximo de dominio. aplica consistencia de arco a todo el CSP de forma sistematica. Piensa en el como un robot que revisa todas las conexiones del grafo, una por una, hasta que no queda nada por eliminar.

Los 6 pasos del algoritmo

1

Crear una cola con todos los arcos (Xi, Xj) del CSP. Cada restriccion binaria genera dos arcos: (Xi→Xj) y (Xj→Xi).

2

Sacar un arco (Xi, Xj) de la cola.

3

Para cada valor v en D(Xi): si no existe ningun valor en D(Xj) compatible con v, eliminar v de D(Xi).

4

Si D(Xi) cambio → agregar a la cola todos los arcos (Xk, Xi) donde Xk es vecino de Xi (excepto Xj).

5

Si D(Xi) queda vacio → el CSP no tiene solucion. Parar.

6

Repetir desde paso 2 hasta que la cola este vacia.

Complejidad: O(e·d³) donde e = numero de arcos y d = tamanio maximo de dominio. En la practica es muy rapido.

Pseudocodigo

AC-3 recibe el CSP completo. REVISE es la funcion auxiliar que revisa un solo arco y devuelve true si elimino algun valor.
function AC-3(csp): cola ← todos los arcos del CSP while cola no vacia: (Xi, Xj) ← sacar de cola if REVISE(Xi, Xj): if D(Xi) == ∅: return false for Xk in vecinos(Xi) - {Xj}: agregar (Xk, Xi) a cola return true function REVISE(Xi, Xj): revised ← false for v in D(Xi): if no ∃ w ∈ D(Xj) compatible con v: eliminar v de D(Xi) revised ← true return revised

¿Por que re-encolar? Porque si D(Xi) se reduce, puede que algun valor de un vecino Xk ya no tenga soporte en Xi. Hay que volver a verificar.

Bloque C · Algoritmo de Busqueda #1

Backtracking: la busqueda basica para CSP

Analogia: Resolver un CSP con backtracking es como resolver un laberinto: avanzas por un camino, y si llegas a un callejon sin salida, retrocedes al ultimo cruce y pruebas otro camino. La diferencia con fuerza bruta es que no exploras caminos que ya sabes que fallan.

¿Como funciona paso a paso?

El backtrackingBacktracking para CSP|||Algoritmo de busqueda en profundidad (DFS) que asigna valores a las variables una a la vez. Despues de cada asignacion, verifica si es consistente con las restricciones. Si una asignacion parcial viola alguna restriccion, retrocede (backtrack) a la variable anterior y prueba otro valor. asigna valores una variable a la vez y verifica inmediatamente:

1 Asigna A = Rojo → ¿consistente con restricciones? ✓ Si
2 Asigna B = Rojo → ¿A≠B? ✗ Falla! (ambos rojos)
↩ BACKTRACK: volver a B, probar siguiente valor
3 Asigna B = Verde → ¿A≠B? Rojo≠Verde ✓ Si
4 Asigna C = ... → continua hasta completar

En Prolog (backtracking natural)

Buena noticia: Prolog ya hace backtracking automaticamente. Su motor de inferencia ES un algoritmo de backtracking. Solo necesitas expresar las restricciones y Prolog busca la solucion solo.
% Definir dominio dominio(rojo). dominio(verde). dominio(azul). % Resolver: asignar y verificar resolver(A,B,C) :- dominio(A), % asigna A dominio(B), % asigna B A \= B, % ¿A≠B? si no, backtrack dominio(C), % asigna C A \= C, % ¿A≠C? B \= C. % ¿B≠C? ?- resolver(A, B, C). % A=rojo, B=verde, C=azul

Ventaja de Prolog: el backtracking viene gratis — es parte del motor de inferencia. Pero tiene un problema: solo detecta fallos cuando llega a la restriccion. ¿Se puede hacer mejor?

Bloque C · Algoritmo de Busqueda #2

Forward Checking: mirar hacia adelante

El problema del backtracking simple

El backtracking solo detecta fallos cuando intenta asignar una variable. Puede avanzar mucho antes de darse cuenta de que una asignacion anterior ya causaba un problema inevitable.

Analogia: Es como conducir hacia un callejon sin salida y solo darte cuenta cuando llegas al fondo y no puedes avanzar. Forward checking es como leer las senales de trafico antes de entrar — si dice "sin salida", ni te metes.

¿Que mejora Forward Checking?

Despues de cada asignacion, propagaPropagacion|||Proceso de eliminar valores del dominio de variables futuras que son inconsistentes con la asignacion actual. Si asigno A=rojo y la restriccion dice A≠B, entonces elimino "rojo" del dominio de B inmediatamente, sin esperar a que B sea asignada. los efectos: elimina valores imposibles de las variables no asignadas. Si algun dominio queda vacio, retrocede inmediatamente sin desperdiciar esfuerzo.

Ejemplo completo paso a paso

Variables: A, B, C. Dominios: {R, V, Az}. Restricciones: A≠B, A≠C, B≠C.

Estado inicial

D(A)={R,V,Az}   D(B)={R,V,Az}   D(C)={R,V,Az}

Paso 1: Asignar A = R → propagar a vecinos

D(B)={R,V,Az}   D(C)={R,V,Az}

R eliminado de B y C porque A≠B y A≠C

Paso 2: Asignar B = V → propagar a vecinos

D(C)={R,V,Az} = {Az}

V eliminado de C porque B≠C. ¡C solo tiene una opcion!

Paso 3: C solo puede ser Az → asignar directamente

Solucion: A=Rojo, B=Verde, C=Azul

¿Ves la diferencia? El backtracking simple habria probado A=R, B=R (falla), B=V, C=R (falla), C=V (falla), C=Az (exito) — 6 pasos. Forward checking lo resolvio en 3 pasos sin fallos porque propago las restricciones.

Bloque C · La Herramienta Poderosa

CLP(FD): Restricciones sobre Dominios Finitos

¿Por que existe CLP(FD)? Ya vimos que Prolog hace backtracking, pero con el enfoque "generate-and-test" es lento porque prueba todo antes de verificar. CLP(FD)CLP(FD) — Constraint Logic Programming over Finite Domains|||Libreria de SWI-Prolog que implementa un solver de restricciones sobre dominios finitos (numeros enteros). Combina propagacion de restricciones + forward checking + heuristicas. En vez de generar-y-probar, propaga restricciones automaticamente reduciendo dominios. Es dramaticamente mas eficiente que el backtracking simple. es una libreria de SWI-Prolog que implementa automaticamente todo lo que vimos: consistencia de arco, forward checking y heuristicas. ¡Es como pasar de una bicicleta a un auto deportivo!

Operadores CLP(FD) — Tu vocabulario nuevo

Ejemplo simple: 3 variables diferentes

Comparemos el enfoque viejo (generate-and-test) con CLP(FD). El codigo CLP(FD) es mas corto, mas claro, y miles de veces mas rapido para problemas grandes.
% ANTES (generate-and-test, lento): viejo(A,B,C) :- member(A,[1,2,3]), member(B,[1,2,3]), member(C,[1,2,3]), A \= B, A \= C, B \= C. % AHORA (CLP(FD), rapido): :- use_module(library(clpfd)). nuevo(A,B,C) :- [A,B,C] ins 1..3, all_different([A,B,C]), label([A,B,C]).

¿Que hace label/1? Es el "boton de inicio" del solver. Primero defines variables, dominios y restricciones — CLP(FD) propaga restricciones y reduce dominios. Luego label busca una asignacion concreta que satisfaga todo. Sin label, Prolog solo te muestra los dominios reducidos, no valores concretos.

Bloque C · CLP(FD) en Accion

SEND + MORE = MONEY: el puzzle clasico

Este es un puzzle criptaritmeticoCriptaritmetica|||Tipo de puzzle donde letras representan digitos y forman una ecuacion aritmetica. Cada letra es un digito diferente (0-9). El reto es descubrir que digito corresponde a cada letra para que la ecuacion sea correcta.: cada letra representa un digito diferente (0-9), y la suma debe ser correcta. Resolverlo a mano tomaria horas; con CLP(FD), milisegundos.

Codigo linea por linea

% Linea 1: cargar la libreria CLP(FD) :- use_module(library(clpfd)). % Linea 2: definir el predicado con 8 variables sendmore(Digits) :- % Linea 3: nombrar las 8 variables Digits = [S,E,N,D,M,O,R,Y], % Linea 4: dominio = digitos 0 a 9 Digits ins 0..9, % Linea 5: cada letra = digito diferente all_different(Digits), % Linea 6: S y M no pueden ser 0 % (no escribimos 0567 + 0085) S #\= 0, M #\= 0, % Linea 7-9: la ecuacion aritmetica 1000*S + 100*E + 10*N + D + 1000*M + 100*O + 10*R + E #= 10000*M + 1000*O + 100*N + 10*E + Y, % Linea 10: ¡resolver! label(Digits).

¿Que pasa internamente?

1. Establece dominios iniciales

Todas las variables: {0..9}

2. Propaga restricciones unarias

S #\= 0 → D(S)={1..9}, M #\= 0 → D(M)={1..9}

3. Propaga la ecuacion

SEND + MORE = MONEY → M solo puede ser 1 (carry)

La suma de dos numeros de 4 digitos max = 19999, asi que M=1

4. label() asigna y verifica

S=9, E=5, N=6, D=7, M=1, O=0, R=8, Y=2

9567 + 1085 = 10652 ✓

SEND + MORE = MONEY verificado

Sin CLP(FD), habria que probar 10^8 = 100 millones de combinaciones. Con CLP(FD), la propagacion reduce esto a unas pocas docenas de pruebas.

Bloque C · Heuristicas de Seleccion

Heuristicas: ¿en que orden asignar?

El orden en que elegimos variables y valores puede hacer una diferencia enorme en la velocidad. Piensa en un Sudoku: ¿empezarias por una celda con 8 opciones o por una con solo 1 opcion? La intuicion dice que por la mas restringida — y la teoria lo confirma.

MRVMRV — Minimum Remaining Values|||Heuristica que selecciona la variable con el menor numero de valores restantes en su dominio. La logica: si una variable solo tiene 2 opciones, es mas probable que falle pronto → detectamos fallos antes y podamos mas ramas del arbol de busqueda. Se conoce como "fail-first".

¿Que variable asignar primero?

Elegir la variable con menos valores posibles en su dominio.

D(A) = {R, V, Az} → 3 opciones D(B) = {V, Az} → 2 opciones D(C) = {Az} → 1 opcion ← primero
Logica "fail-first": si C solo tiene 1 opcion, asignalo ya. Si falla, lo sabes inmediatamente y ahorras tiempo.

GradoHeuristica de Grado|||Cuando hay empate en MRV, elige la variable que tiene mas restricciones con variables no asignadas. La idea: asignar primero las variables mas "conectadas" reduce mas el espacio de busqueda para las demas. Mas conexiones = mas propagacion.

Desempate si MRV da empate

Elegir la variable con mas restricciones sobre variables no asignadas.

A → 3 restricciones con no-asignadas B → 4 restricciones ← primero C → 2 restricciones
Logica: la variable mas conectada "propaga" mas informacion al ser asignada.

LCVLCV — Least Constraining Value|||Heuristica para el orden de los valores (no de las variables). Elige primero el valor que elimina menos opciones de las variables vecinas. La logica: maximizar la flexibilidad futura — dejar mas opciones abiertas para las variables que aun no se asignaron.

¿Que valor probar primero?

Preferir el valor que deja mas opciones abiertas para los vecinos.

A=R → elimina R de 3 vecinos A=V → elimina V de 2 vecinos ← mejor A=Az → elimina Az de 3 vecinos
Logica "succeed-first": maximizar la probabilidad de encontrar solucion sin backtrack.

Resumen: MRV y Grado deciden que variable asignar primero (fail-first). LCV decide que valor probar primero (succeed-first). Juntas, estas heuristicas aceleran dramaticamente la busqueda. CLP(FD) las aplica automaticamente con label/1.

Bloque D · Ejercicio Practico

Ejercicio: Asignacion de Horarios con CSP

El objetivo: Asignar 4 cursos a franjas horarias de modo que no haya conflictos. Usa CLP(FD) para modelar las restricciones y dejar que Prolog encuentre la solucion automaticamente.

Los datos del problema

Cursos (= variables)

Matematicas (M), Fisica (F), Quimica (Q), Informatica (I)

Franjas horarias (= dominio)

1 = 8:00-10:00  ·  2 = 10:00-12:00  ·  3 = 13:00-15:00

Restricciones (= reglas)

1. M y F no en la misma franja (mismo profesor) → binaria: M #\= F 2. Q e I no en la misma franja (mismos alumnos) → binaria: Q #\= I 3. M debe ser en la manana (franja 1 o 2) → unaria: M in 1..2

Guia paso a paso para construirlo

1 Cargar CLP(FD)

Escribe :- use_module(library(clpfd)). al inicio del archivo.

2 Definir variables y dominios

Crea Vars = [M, F, Q, I] y luego Vars ins 1..3 para asignar dominios.

3 Expresar restricciones

M #\= F (binaria), Q #\= I (binaria), M in 1..2 (unaria).

4 Resolver con label

label(Vars) le dice a CLP(FD) que busque una asignacion que satisfaga todo.

Bloque D · Solucion Completa

Solucion: Horario con CLP(FD)

Codigo completo con explicacion

horario(M,F,Q,I) recibe 4 variables (una por curso). Define dominios (franjas 1-3), aplica las 3 restricciones, y usa label para encontrar la asignacion. Todo en 7 lineas utiles.
:- use_module(library(clpfd)). horario(M, F, Q, I) :- % 1. Variables y dominios Vars = [M, F, Q, I], Vars ins 1..3, % 2. Restricciones binarias M #\= F, % mismo profesor Q #\= I, % mismos alumnos % 3. Restriccion unaria M in 1..2, % manana % 4. Resolver label(Vars). ?- horario(M, F, Q, I). % M=1, F=2, Q=1, I=2 % (primera solucion encontrada)

¿Que hace Prolog internamente?

1. Establece dominios

D(M)=D(F)=D(Q)=D(I) = {1,2,3}

2. Aplica restriccion unaria

M in 1..2 → D(M) = {1,2} (se elimina 3)

3. Propaga restricciones binarias

M #\= F: si M=1, F no puede ser 1

4. label() busca solucion

M=1, F=2, Q=1, I=2 → ¿todas ok? ✓

 Verificacion manual: M=1(8am), F=2(10am) → M≠F ✓. Q=1(8am), I=2(10am) → Q≠I ✓. M=1 → M ∈ {1,2} ✓. ¡Todo correcto!

Reto extra: Agrega la restriccion F #\= Q (Fisica y Quimica no en la misma franja). ¿Cuantas soluciones quedan? Ver respuestaRespuesta|||Con la restriccion extra F #\= Q, las soluciones se reducen. Puedes verificar con findall(S, horario(M,F,Q,I), Soluciones) y contar la lista.

Cierre de Sesion

Las 3 ideas que debes llevarte hoy

Idea 1: Modelar antes de resolver

Un CSP se define con tres cosas: variables, dominios y restricciones. Modelar bien el problema es el 80% del trabajo — el solver hace el resto. Piensa en la analogia de la boda: primero defines las reglas, luego el sistema asigna los asientos.

Idea 2: Propagar, no adivinar

La consistencia de arco (AC-3) y el forward checking eliminan opciones imposibles antes de buscar. Esto hace la diferencia entre milisegundos y horas de computo. Es como leer las senales de transito antes de manejar.

Idea 3: CLP(FD) = CSP en Prolog

Con use_module(library(clpfd)) tienes un solver completo: dominios con ins, restricciones con #= / #\=, y resolucion con label. Todo en pocas lineas — el solver aplica AC-3, forward checking y heuristicas automaticamente.
